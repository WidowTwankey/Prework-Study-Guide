HTML

For the creation of the webpage; content and structure

  • Text, image, header and list elements are used to add content to a webpage
  • The 'head' element contains information about the webpage
  • The 'body' element represents the visible content shown to the user
  • Semantic elements describe the content and role of what they do on a webpage
  • HTML is fun

CSS

For styling the webpage and

  • A margin indicates how much space we want around the outside of an element
  • A border lies within the margin
  • A padding, inside the border, indicates how much space we want around the content inside an element
  • You can assign colours by using the hexidecimal code (or hex #), which you can find on the HTML colour codes website (my favourite colour is close to #68FFD8)

Git

For control of the terminal and interaction with GitHub

  • Make sure you're on the correct branch before editing anything, or checking out to the right branch will wipe all of your progress
  • "git status" checks what branch of the repo you're currently on
  • "git add -A" adds the changes you've made to the current branch?
  • "git commit -m "added HTML to project""" commits the changes with the message that you've added HTML
  • "git pull origin main" pulls the latest version of main branch
  • "git push origin feature/add-html" pushes up all of the changes
  • "git checkout main" moves you to the main branch
  • "git checkout -b creepy-crawlies" adds and puts you on a new branch called 'creepy-crawlies'

Javascript

The most complex of the four topics, for interaction in the browser

  • A variable is a named container that allows us to store data in our code, e.g. var topics
  • Control flow is the order in which a computer executes code in a script. In JS, it runs like it reads, top to bottom.
